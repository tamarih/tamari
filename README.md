# קטלוג מוצרי סובלימציה - מדריך הקמה

מערכת קטלוג + ניהול מחירים מבוססת HTML/JS + Supabase.
המערכת מחשבת אוטומטית: **מחיר ספק → +18% מע"מ → +30% רווח**.

---

## מבנה הפרויקט

```
catalog-site/
├── index.html              ← עמוד הקטלוג הציבורי
├── admin.html              ← עמוד הניהול (דורש התחברות)
├── schema.sql              ← סכמת מסד הנתונים
├── css/styles.css          ← עיצוב
├── js/
│   ├── supabase-client.js  ← הגדרות חיבור + חישובי מחיר
│   ├── catalog.js          ← לוגיקת קטלוג
│   └── admin.js            ← לוגיקת ניהול
├── data/
│   ├── seed-products.json  ← 42 מוצרי דוגמה (מה-PDF)
│   └── seed-products.sql   ← SQL לטעינה ל-Supabase
└── README.md
```

## שלב 1: הקמת Supabase (5 דקות)

1. נכנסים ל-[supabase.com](https://supabase.com) ופותחים חשבון חינמי.
2. **New Project** - נותנים שם (`zikit-catalog`), בוחרים אזור (Frankfurt), קובעים סיסמת מסד נתונים.
3. ממתינים 1-2 דקות עד שהפרויקט נבנה.
4. נכנסים ל-**SQL Editor** (סרגל ימני) → **New Query**.
5. מעתיקים את **כל התוכן** של `schema.sql` → מדביקים → **Run**.
6. New Query נוסף → מעתיקים את `data/seed-products.sql` → **Run**. (טעינת 42 מוצרי דוגמה.)

## שלב 2: יצירת משתמש ניהול

1. **Authentication** (סרגל ימני) → **Users** → **Add user** → **Create new user**.
2. אימייל + סיסמה חזקה. שומרים.
3. ✅ זה משתמש הניהול - רק הוא יוכל להוסיף/לערוך מוצרים.

## שלב 3: חיבור האתר ל-Supabase

1. ב-Supabase: **Project Settings** → **API**.
2. מעתיקים **2 דברים**:
   - `Project URL` (נראה כך: `https://xxxxxx.supabase.co`)
   - `anon public` key (מחרוזת ארוכה שמתחילה ב-`eyJ...`)
3. פותחים את הקובץ `js/supabase-client.js`.
4. מחליפים את 2 השורות בראש הקובץ:
   ```js
   const SUPABASE_URL  = "https://xxxxxx.supabase.co";
   const SUPABASE_ANON = "eyJ....שלך כאן....";
   ```
5. שומרים.

## שלב 4: בדיקה מקומית

פתח את `index.html` בדפדפן (דאבל-קליק). אתה אמור לראות את 42 מוצרי הדוגמה.
פתח את `admin.html`, התחבר עם המשתמש שיצרת ב-Authentication, ונסה להוסיף מוצר.

---

## שלב 5: העלאה ל-Netlify (חינמי, 2 דקות)

1. נכנסים ל-[app.netlify.com](https://app.netlify.com) ופותחים חשבון.
2. **Sites** → **Add new site** → **Deploy manually**.
3. גוררים את **כל התיקייה** `catalog-site` לתוך החלון.
4. ✅ קיבלת כתובת `https://xxx-xxx.netlify.app`.
5. אופציונלי: **Domain settings** → לחבר דומיין משלך.

> **חידוש האתר:** בכל שינוי בקבצים - גוררים את התיקייה שוב לאותה Site → "Deploys".

## חישוב המחירים

המחירים מחושבים **אוטומטית** מתוך מחיר ספק יחיד:

| שדה | חישוב | דוגמה |
|---|---|---|
| מחיר ספק | קלט מהמשתמש | ₪10.00 |
| מחיר כולל מע"מ | × 1.18 | ₪11.80 |
| מחיר מכירה +30% | × 1.30 | ₪15.34 |
| רווח בש"ח | מכירה - כולל מע"מ | ₪3.54 |

לעדכון אחוז המע"מ או הרווח לכל המוצרים - שינוי הקבועים בראש `js/supabase-client.js`:
```js
const VAT_RATE    = 18;
const MARGIN_RATE = 30;
```

ה-PDF המקורי מכיל 713 עמודים = מאות מוצרים. נטענו 42 כדוגמה.

- אפשרות ב': חילוץ אצוות נוספות מה-PDF + הוספה לקובץ `data/seed-products.sql` (ניתן לחזור אליי בכל שלב).

## בעיות נפוצות

- **המוצרים לא נטענים בקטלוג:** בדקו שהמפתחות ב-`js/supabase-client.js` נכונים, ושהרצתם את `schema.sql`.
- **לא מצליחים להתחבר לניהול:** ודאו שיצרתם משתמש ב-Authentication → Users (לא רק רשמתם!).
- **העלאת תמונה נכשלת:** ודאו שהרצתם את כל ה-`schema.sql` (כולל יצירת bucket `product-images`).
- **שינוי במחיר ספק לא מתעדכן בכל המקומות:** רענון העמוד - החישוב הוא תמיד מ-`supplier_price` החי במסד.
